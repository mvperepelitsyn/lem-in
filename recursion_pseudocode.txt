
Рекурсия - управляющая ф-ция поиска - запускается в цикле, пока не вернёт SUCCESS:

1.	Проверяем есть ли свободная связь для нового маршрута

2.	Условие выхода: если кол-во найденных путей == кол-ву муравьёв || уже нет свободных связей для новых путей из старта
	--> возвращаем SUCCESS

3.	Запускаем поиск самого короткого пути
	|
	 -> Если SUCCESS --> добавляем созданный путь, фиксируем сет цикла
	|	|	если кол-во шагов найденного сета меньше чем у предыдущего -> запускаем рекурсивный вызов для поиска следующего пути
	|	|	|
	|	|	 -> если больше --> возвращаем SUCCESS
	|	|
	|	 -> Если рекурсия возвращает SUCCESS --> возвращаем SUCCESS
	|	|
	|	 -> if else (если FAILURE)
	|		|
	|		 -> возвращаем FAILURE
	|
	 -> Если мы наткнулись на использованную комнату => возвращаем FAILURE

Функция поиска самого широкого пути сама запускает процесс удаления связи и стерает информацию конкретного маршрута в графе.
